# Security Policy

## Supported versions
Security fixes are applied on the latest `main` branch and latest release tag.

## Reporting a vulnerability
Please do **not** open a public issue for security vulnerabilities.

Send a private report to the repository owner through GitHub Security Advisories:
- Go to the repository
- Open **Security**
- Select **Report a vulnerability**

If GitHub Security Advisories are not available, open a private contact request through `SUPPORT.md` and include:
- Affected version/commit
- Reproduction steps
- Impact assessment
- Suggested mitigation (if known)

## Disclosure process
- Acknowledge report within 5 business days.
- Provide a triage status and severity.
- Coordinate fix, test, and release notes.
- Credit reporter if requested.
